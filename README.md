# Insignia-Subnet
The Insignia subnet utilizes the bittensor network to structure a decentralized quantitative trading firm. Researcher miners compete to produce predictive machine learning models, scored by validators against a proprietary tick-by-tick benchmark dataset, while trading-operations miners build execution strategies that inference those models. Rather than ranking the two skill sets separately, Insignia uses a **single incentive mechanism** that matches a researcher and a trader into a `(model, strategy)` pair and evaluates them together — like individuals in a genetic algorithm. Validators query one researcher miner and one trader miner per pair, score the pair on the combined model-quality and live-trading metrics, and select/breed pairs across generations with an NSGA-II-style multi-objective genetic algorithm. This produces battle-tested model+strategy pairs eligible for deployment by the subnet owner's prop trading firm. The mechanism is hardened against miner collusion, miner-validator collusion, and latency arbitrage via chain-seeded pairing, variance-penalized marginal-contribution credit, and commit-reveal. See [subnet/docs/PAIRING_MECHANISM.md](subnet/docs/PAIRING_MECHANISM.md).
Multiple strategies are used to economically support inscentives and investors. The firm uses profits from the top deployed models for alpha token buy backs in order to counter sell pressure from miners. API access to the firm's top real time strategies are accessable to holders of the alpha token, suppessing sell pressure. Any external investments from quant shops or investors, into the fund, amplifies inflows and inscentives.
The subnet utilizes decentralization and open source to diversify strategies usind predictive modeling for high frequency trading, and incentivise competition to stay current with market state transitions.
