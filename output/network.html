Step 1 · Layer 1

L1 Miner Trains Model

Miners compete to build the best ML model for short-horizon crypto market prediction. Each miner trains independently using their own data, features, algorithms, and hyperparameters.

Step 2

Model Submission

The trained model is packaged into an L1ModelSubmission synapse and sent to validators over the Bittensor network. This is the only interface between miners and the scoring system.

model_artifact: bytes  (serialized sklearn pipeline)
features_used: List[str]  (from public registry)
target_instrument: "BTC-USDT-PERP" | "ETH-USDT-PERP" | ...
target_horizon: int  (minutes)
hyperparams: {n_estimators, max_depth, ...}
generalization_gap: float  (self-reported)
Step 3 · Gate

Anti-Gaming Checks

Before any model is scored, it passes through three defensive gates. Each blocks a specific attack vector. Failure at any gate rejects the submission entirely.

✓
Rate Limiting — Max 1 submission per miner per epoch (24h). Blocks brute-force spam attacks.
✓
Duplicate Detection — SHA-256 fingerprint of model artifact. Blocks exact copies of other miners' models.
✓
Clone Detection — Prediction correlation analysis. If two models correlate >95%, they split rewards. Removes free-riding incentive.
Step 4

Step 5 · Bridge

Model Promotion to L2

The top-N scoring models are promoted to the Layer 2 pool, where L2 miners can use them to build real trading strategies. Not every model qualifies — three criteria must be met.

◆
Generalization gap ≤ 40% — Model must not be severely overfit.
◆
≥ 2 consecutive positive epochs — One good epoch could be luck. Two shows persistence.
◆
Score decay ≤ 20% — Models deteriorating rapidly are not promoted. Stale models expire after 5 unused epochs.

Step 8 · Feedback Loop

Cross-Layer Feedback

L2 trading performance retroactively adjusts L1 model scores in the next epoch. This closes the simulation-to-reality gap — a model is only "good" if its deployed strategies actually make money.

+15%
Bonus multiplier on next L1 epoch
When L2 strategy score > 0.6 (after 3+ epochs of evidence)
−10%
Penalty on next L1 epoch
When L2 strategy score < 0.3 (despite good simulation scores)
Step 8 feeds back into Step 4 — L2 results adjust L1 scores in the next epoch, creating a continuous improvement loop.
